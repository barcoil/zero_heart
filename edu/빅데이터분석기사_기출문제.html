Q45. Sqoop과 Flume의 특징을 비교한 내용으로 옳은 것은?

  3. Sqoop과 Flume 모두 실시간 스트리밍 처리를 주목적으로 한다

Q47. RDBMS와 NoSQL의 비교 설명으로 옳지 않은 것은?

  1. RDBMS는 ACID 특성을 보장하고, NoSQL은 주로 BASE 특성을 따른다
  2. RDBMS는 수직 확장(Scale-up), NoSQL은 수평 확장(Scale-out)에 유리하다
  3. NoSQL은 고정 스키마(Fixed Schema)를 사용하여 데이터 일관성을 높인다
  4. RDBMS는 정형 데이터, NoSQL은 정형·반정형·비정형 데이터 모두 처리 가능하다

✅ 정답: 3번 | NoSQL의 특징 중 하나가 유연한 스키마(Schema-less 또는 Dynamic Schema). 고정 스키마는 RDBMS의 특징

Q48. 데이터 웨어하우스(Data Warehouse)와 데이터 마트(Data Mart)의 차이로 옳은 것은?

  2. 데이터 웨어하우스는 전사적 통합 저장소이고, 데이터 마트는 특정 주제·부서에 맞게 추출한 소규모 저장소이다
  3. 데이터 웨어하우스와 데이터 마트는 동일한 개념이다
  4. 데이터 마트는 OLTP 시스템이고, 데이터 웨어하우스는 실시간 트랜잭션 처리 시스템이다

✅ 정답: 2번 | DW=전사 통합(대규모, 주제지향·통합·시간가변·비휘발성), DM=DW에서 추출한 특정 부서/주제용 소규모 저장소

Q49. OLAP(Online Analytical Processing)에 대한 설명으로 옳지 않은 것은?

  3. 롤업(Roll-up)은 세부 데이터를 집계하여 상위 수준으로 올라가는 연산이다
  4. 실시간 트랜잭션 처리(OLTP) 시스템과 동일한 목적으로 사용된다

✅ 정답: 4번 | OLAP=분석·의사결정 지원(읽기 중심), OLTP=실시간 트랜잭션 처리(쓰기 중심). 목적이 다름

Q50. 분석 조직 유형에 대한 설명으로 옳은 것은?

  3. 혼합형 구조는 집중형과 분산형을 결합하여 전담 조직과 현업 부서 모두에 분석 인력을 배치한다

✅ 정답: 3번 | 집중형(CoE)=전담 중앙집중, 분산형=현업 각 부서 배치, 혼합형=두 방식 결합(가장 이상적). 분산형은 현업 친화적이나 중복 비효율 발생 가능

Q53. 전진선택법(Forward Selection)에 대한 설명으로 옳은 것은?

  1. 모든 변수를 포함한 상태에서 시작하여 기여도가 낮은 변수를 하나씩 제거한다
  2. 변수가 없는 상태에서 시작하여 설명력이 가장 높은 변수를 하나씩 추가한다
  3. 변수 추가와 제거를 반복하며 최적 변수 집합을 선택한다
  4. 가능한 모든 변수 조합을 탐색하여 최적 모형을 선택한다

✅ 정답: 2번 | 전진선택법: 변수 없음→AIC/F통계량 기준 가장 유의한 변수 순차 추가. 후진제거=전체→제거, 단계적=추가+제거 반복

Q54. 변수 선택 방법 중 후진제거법(Backward Elimination)의 특징으로 옳지 않은 것은?

  1. 모든 변수를 포함한 완전 모형에서 시작한다
  2. 유의하지 않은 변수를 하나씩 제거해 나간다
  3. 전진선택법보다 계산량이 적다
  4. p값 또는 AIC를 기준으로 제거할 변수를 선택한다

✅ 정답: 3번 | 후진제거법은 전체 변수로 시작하므로 초기 계산량이 전진선택법보다 많음. 변수 수가 많을수록 초기 모형의 계산 비용이 큼

Q55. 단계적 선택법(Stepwise Selection)에 대한 설명으로 옳은 것은?

  1. 전진선택만 수행하여 변수를 추가한다
  2. 후진제거만 수행하여 변수를 제거한다
  3. 변수 추가와 제거를 반복하며, 이미 추가된 변수도 유의하지 않으면 제거할 수 있다
  4. 모든 가능한 변수 조합을 탐색하는 완전 탐색법이다

✅ 정답: 3번 | 단계적 선택법: 전진선택+후진제거를 반복. 이미 추가된 변수도 후속 변수 추가 후 유의하지 않으면 제거 가능. 전진선택법의 단점 보완

Q56. Shapiro-Wilk 검정에 대한 설명으로 옳은 것은?

  2. 표본이 정규분포를 따르는지 검정한다

Q57. Shapiro-Wilk 검정의 귀무가설(H₀)로 옳은 것은?

  2. 데이터는 정규분포를 따른다

Q58. 표본 추출 방법 중 층화추출법(Stratified Sampling)에 대한 설명으로 옳은 것은?

  2. 모집단을 동질적인 층(Stratum)으로 나누고, 각 층에서 독립적으로 표본을 추출한다
  3. 첫 번째 표본을 무작위로 선택하고 이후 일정 간격으로 표본을 추출한다
  4. 모집단에서 완전히 무작위로 표본을 추출한다

✅ 정답: 2번 | 층화추출: 모집단을 동질적 층으로 분류→각 층에서 비례·균등 추출. 군집추출=일부 군집 선택 후 전수조사, 계통추출=일정 간격 추출

Q60. 군집추출법(Cluster Sampling)과 층화추출법(Stratified Sampling)의 차이로 옳은 것은?

  2. 군집추출은 이질적인 군집들 중 일부를 선택하고, 층화추출은 동질적인 층 모두에서 추출한다
  3. 두 방법 모두 동일한 방식으로 표본을 추출한다
  4. 층화추출이 군집추출보다 항상 표본 오차가 크다

✅ 정답: 2번 | 군집추출: 이질적인 군집(내부 이질, 군집 간 동질) 중 일부 군집 선택→전수/추출. 층화추출: 동질적 층(층 내 동질) 모두에서 추출→정밀도 높음

  2. 서포트 벡터(Support Vector)는 결정 경계에 가장 가까운 데이터 포인트들이다
  3. 커널 트릭(Kernel Trick)을 사용하여 비선형 분류가 가능하다
  4. SVM은 비지도 학습 알고리즘으로 레이블 없이 군집을 형성한다

✅ 정답: 4번 | SVM은 지도 학습(Supervised Learning) 기반 분류/회귀 알고리즘. 비지도 학습이 아님

Q63. SVM에서 소프트 마진(Soft Margin)과 하드 마진(Hard Margin)에 관한 설명으로 옳은 것은?

  1. 하드 마진은 일부 오분류를 허용하여 더 유연한 결정 경계를 만든다
  2. 소프트 마진은 슬랙 변수(Slack Variable)를 도입하여 완벽한 분리가 어려운 경우에도 적용 가능하다
  3. 소프트 마진은 선형 분리 불가능한 데이터에 사용할 수 없다
  4. 하드 마진과 소프트 마진은 동일한 개념이다

✅ 정답: 2번 | 소프트 마진: 슬랙 변수(ξ) 도입→일부 오분류 허용→노이즈에 강인. 하드 마진: 완벽한 선형 분리 가정→현실 데이터에 과적합 위험

Q64. SVM에서 커널 함수(Kernel Function)를 사용하는 이유로 가장 적절한 것은?

  1. 학습 속도를 높이기 위해
  2. 저차원 공간에서 선형 분리가 불가능한 데이터를 고차원 공간에서 분리하기 위해
  3. 과적합을 방지하기 위해
  4. 클래스 불균형 문제를 해결하기 위해

✅ 정답: 2번 | 커널 트릭: 명시적 고차원 변환 없이 내적 계산으로 고차원 매핑 효과. 대표 커널: 선형, 다항식, RBF(가우시안), 시그모이드

Q65. 연관 분석에서 신뢰도(Confidence)의 계산식으로 옳은 것은?

  1. P(A ∩ B)
  2. P(A ∩ B) / P(A)
  3. P(A ∩ B) / (P(A) × P(B))
  4. P(A ∩ B) / P(B)

✅ 정답: 2번 | 신뢰도(Confidence) = P(A∩B)/P(A) = P(B|A). 지지도=P(A∩B), 향상도=P(A∩B)/(P(A)×P(B)) = 신뢰도/P(B)

Q66. 연관 분석에서 향상도(Lift)가 1보다 클 때의 의미로 옳은 것은?

  1. A와 B는 서로 독립적인 관계이다
  2. A가 있을 때 B가 발생할 확률이 B의 전체 발생 확률보다 낮다
  3. A와 B는 음의 연관 관계이다
  4. A가 있을 때 B가 발생할 확률이 B의 전체 발생 확률보다 높다 (양의 연관)

✅ 정답: 4번 | 향상도>1: 양의 연관(A→B 시너지), =1: 독립, <1: 음의 연관. 향상도=신뢰도/P(B)=P(A∩B)/(P(A)×P(B))

Q67. 다음 장바구니 데이터에서 {우유}→{빵} 규칙의 지지도(Support)를 구하면? (전체 거래 10건 중 우유와 빵이 함께 포함된 거래 4건, 우유만 포함된 거래 2건)

  1. 0.2
  2. 0.4
  3. 0.6
  4. 0.67

✅ 정답: 2번 | 지지도 = (우유∩빵 거래 수)/전체 거래 수 = 4/10 = 0.4

Q68. ARIMA(p,d,q) 모형에서 각 파라미터의 의미로 옳은 것은?

  1. p=이동평균 차수, d=차분 횟수, q=자기회귀 차수
  2. p=자기회귀 차수, d=차분 횟수, q=이동평균 차수
  3. p=계절성 주기, d=추세, q=잔차
  4. p=차분 횟수, d=자기회귀 차수, q=이동평균 차수

✅ 정답: 2번 | ARIMA(p,d,q): p=AR(자기회귀) 차수, d=차분(I: Integrated) 횟수, q=MA(이동평균) 차수. 비정상 시계열→d회 차분→정상화

Q69. 시계열 분석에서 정상성(Stationarity) 조건으로 옳은 것은?

  1. 시간에 따라 평균이 변하고 분산이 일정한 시계열
  2. 평균, 분산, 자기공분산이 시간에 관계없이 일정한 시계열
  3. 추세(Trend)와 계절성(Seasonality)이 존재하는 시계열
  4. 분산은 변해도 되지만 평균은 일정해야 하는 시계열

✅ 정답: 2번 | 정상 시계열 조건: ①평균이 일정, ②분산이 일정, ③공분산이 시차에만 의존(시점 무관). ARIMA 적용 전 차분(Differencing)으로 정상화 필요

Q70. KNN(K-최근접 이웃) 알고리즘에서 K값 선택에 대한 설명으로 옳은 것은?

  1. K가 클수록 결정 경계가 복잡해지고 과적합 위험이 높아진다
  2. K=1이면 훈련 데이터에 대해 항상 100% 정확도를 보인다
  3. K값이 클수록 편향(Bias)이 감소하고 분산(Variance)이 증가한다
  4. K는 항상 홀수로 설정해야 한다

✅ 정답: 2번 | K=1: 자기 자신이 가장 가까운 이웃 → 훈련 정확도 100%(과적합). K↑: 편향↑, 분산↓(결정경계 단순). K=짝수는 동률 발생 가능하여 홀수 권장하지만 필수는 아님

Q71. KNN 알고리즘의 단점으로 옳지 않은 것은?

  1. 예측 시 모든 훈련 데이터와 거리를 계산해야 하므로 데이터가 많을수록 느리다
  2. 특성(Feature) 스케일에 민감하므로 정규화/표준화가 필요하다
  3. 모델 학습 단계가 없어 훈련 시간이 매우 짧다(게으른 학습)
  4. 차원이 높아질수록 성능이 향상되는 특성이 있다

✅ 정답: 4번 | KNN은 차원의 저주(Curse of Dimensionality) 문제: 고차원일수록 데이터 밀도 희박→거리 의미 감소→성능 저하

Q72. 연관 규칙 마이닝(Apriori 알고리즘)의 특징으로 옳은 것은?

  1. 하향식(Top-down) 방식으로 빈발 항목집합을 생성한다
  2. 최소 지지도(Minimum Support)를 만족하는 항목집합만 탐색한다
  3. 향상도만을 기준으로 연관 규칙을 선별한다
  4. 순차적 패턴 분석에만 사용된다

✅ 정답: 2번 | Apriori 알고리즘: 상향식(Bottom-up), 최소 지지도 기반 빈발 항목집합 탐색, 최소 신뢰도로 규칙 생성. "비빈발 항목집합의 모든 부분집합도 비빈발" 성질 활용

Q73. 분산팽창요인(VIF, Variance Inflation Factor)에 대한 설명으로 옳지 않은 것은?

  1. 다중공선성을 진단하는 지표이다
  2. VIF > 10이면 심각한 다중공선성 문제가 있다고 판단한다
  3. VIF 값이 1에 가까울수록 다중공선성이 심각하다
  4. VIF가 크면 해당 독립변수는 다른 독립변수들과 강한 선형관계가 있다

✅ 정답: 3번 | VIF=1: 다중공선성 없음, 1<VIF<4: 보통, 4<VIF<10: 다중공선성 존재, VIF>10: 심각. VIF가 1에 가까울수록 독립변수 간 독립적

Q74. 다중공선성(Multicollinearity) 문제를 진단하는 방법으로 옳지 않은 것은?

  1. VIF(분산팽창요인) 계산
  2. 조건지수(Condition Index) 확인
  3. 독립변수 간 상관계수 확인
  4. ROC 곡선 분석

✅ 정답: 4번 | ROC 곡선은 분류 모델의 성능 평가 도구. 다중공선성 진단: VIF, 조건지수(>30 심각), 상관계수 행렬, 고유값 분석

Q75. 다중공선성이 발생했을 때 해결 방법으로 옳지 않은 것은?

  1. 상관성이 높은 변수 중 하나를 제거한다
  2. 주성분 분석(PCA)을 이용해 변수를 변환한다
  3. 릿지 회귀(Ridge Regression)를 사용한다
  4. 훈련 데이터를 더 많이 수집하면 다중공선성이 자동으로 해소된다

✅ 정답: 4번 | 데이터 추가 수집은 다중공선성을 직접 해결하지 않음. 해결법: 변수 제거, 주성분 회귀(PCR), 릿지/라쏘 회귀, 변수 결합

Q76. 편향-분산 트레이드오프(Bias-Variance Tradeoff)에 대한 설명으로 옳은 것은?

  1. 모델 복잡도가 증가하면 편향과 분산이 모두 감소한다
  2. 단순한 모델은 편향이 낮고 분산이 높다
  3. 모델 복잡도가 증가하면 편향은 감소하지만 분산은 증가하는 트레이드오프가 발생한다
  4. 최적 모델은 편향=0, 분산=0인 경우이다

✅ 정답: 3번 | 모델 복잡도↑: 편향↓, 분산↑(과적합). 단순 모델: 편향↑, 분산↓(과소적합). 최적 모델=총 오차(편향²+분산+노이즈) 최소화

Q77. 과소적합(Underfitting) 모델의 특징과 해결 방법으로 옳은 것은?

  1. 훈련 오차가 낮고 검증 오차가 높다 → 더 많은 훈련 데이터 필요
  2. 훈련 오차와 검증 오차 모두 높다 → 더 복잡한 모델 사용
  3. 훈련 오차가 높고 검증 오차가 낮다 → 정규화 적용
  4. 훈련 오차와 검증 오차 모두 낮다 → 모델 단순화 필요

✅ 정답: 2번 | 과소적합: 훈련·검증 오차 모두 높음(고편향). 해결: 더 복잡한 모델, 특성 추가, 정규화 감소. 과적합: 훈련↓ 검증↑(고분산) → 정규화, 데이터 추가

Q78. 하이퍼파라미터(Hyperparameter)와 파라미터(Parameter)의 차이로 옳은 것은?

  1. 파라미터는 훈련 전 사람이 설정하고, 하이퍼파라미터는 학습을 통해 자동으로 결정된다
  2. 하이퍼파라미터는 훈련 전 사람이 설정하고, 파라미터는 학습을 통해 자동으로 결정된다
  3. 파라미터와 하이퍼파라미터는 모두 학습을 통해 결정된다
  4. 하이퍼파라미터는 모델의 가중치(Weight)를 의미한다

✅ 정답: 2번 | 파라미터=학습으로 결정(가중치, 편향 등), 하이퍼파라미터=사람이 사전에 설정(학습률, K값, 트리 깊이, 정규화 강도 등)

Q79. 그리드 서치(Grid Search)와 랜덤 서치(Random Search)의 비교로 옳은 것은?

  1. 그리드 서치는 지정 범위 내 모든 조합을 탐색하고, 랜덤 서치는 무작위 조합을 탐색한다
  2. 랜덤 서치가 그리드 서치보다 항상 더 좋은 하이퍼파라미터를 찾는다
  3. 그리드 서치가 랜덤 서치보다 계산 시간이 항상 짧다
  4. 두 방법 모두 베이지안 최적화(Bayesian Optimization)의 일종이다

✅ 정답: 1번 | 그리드 서치: 모든 격자 조합 탐색(확실하나 계산 비용 높음), 랜덤 서치: 무작위 탐색(빠르고 고차원에 유리). 베이지안 최적화는 별개의 방법

Q80. 베이지안 최적화(Bayesian Optimization)에 대한 설명으로 옳은 것은?

  1. 모든 하이퍼파라미터 조합을 격자 형태로 탐색한다
  2. 이전 시도 결과를 바탕으로 다음에 탐색할 하이퍼파라미터 영역을 스마트하게 선택한다
  3. 완전히 무작위로 하이퍼파라미터를 선택하여 탐색한다
  4. 하이퍼파라미터 최적화가 아닌 모델 구조 탐색에만 사용된다

✅ 정답: 2번 | 베이지안 최적화: 대리 모델(Surrogate Model, 주로 가우시안 프로세스)을 사용하여 이전 결과를 기반으로 탐색 영역을 지능적으로 선택. 그리드·랜덤 서치보다 효율적

Q81. 교차 검증(Cross Validation)과 관련하여 층화 k-겹 교차검증(Stratified k-Fold CV)을 사용하는 이유로 옳은 것은?

  1. 계산 속도를 높이기 위해
  2. 클래스 불균형 데이터에서 각 폴드가 클래스 비율을 유지하도록 보장하기 위해
  3. 회귀 문제에서만 사용 가능한 검증 방법이다
  4. 훈련 데이터를 늘리기 위해

✅ 정답: 2번 | 층화 k-겹 CV: 각 폴드가 원본 데이터의 클래스 비율을 반영. 클래스 불균형(예: 사기 탐지, 암 진단) 데이터에서 편향되지 않은 평가 보장

Q82. 모델 성능 개선을 위한 앙상블 기법 중 부스팅(Boosting)에 대한 설명으로 옳지 않은 것은?

  1. 약한 학습기(Weak Learner)를 순차적으로 결합하여 강한 학습기를 만든다
  2. 이전 모델이 잘못 예측한 샘플에 더 높은 가중치를 부여한다
  3. 배깅(Bagging)에 비해 과적합에 더 취약할 수 있다
  4. 병렬 처리가 가능하여 배깅보다 학습 속도가 빠르다

✅ 정답: 4번 | 부스팅은 순차 학습(병렬 불가). 배깅이 병렬 학습 가능. 부스팅: AdaBoost, Gradient Boosting, XGBoost, LightGBM 등
